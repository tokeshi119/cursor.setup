# モックからAPI設計を行う

## 目的
画面モック/設計からAPI仕様を導出し、OpenAPIで整理する。

## 前提
- フロント主導で要件抽出を進める
- バックエンドはレスポンス最適化の裁量を持つ
- 最終成果はOpenAPI形式

## 手順（7ステップ）
1) 画面要件の分析（表示項目/操作/関連画面）
2) エンドポイント設計（ベースパス/リソース/メソッド/パス）
3) リクエスト仕様（パラメータ/制約）
4) レスポンス構造サンプル
5) 共通仕様（ベースパス/認証/エラー/ステータス）
6) OpenAPI文書化
7) バックエンド引き継ぎ（レビュー依頼/確認事項）

## 出力（最小）
- 画面要件分析
- エンドポイント一覧
- リクエスト/レスポンス例
- 共通仕様（ベースパス/認証/エラー/ステータス）
- OpenAPIファイル
- 引き継ぎメモ
