# Gitワークフロー

このファイルには、Gitを使用した開発ワークフローが定義されています。

## ブランチの確認

`@Branch`を使用して、現在のブランチの変更を確認できます。

## コミットとプッシュ

変更をコミットする前に、必ず`/test`コマンドを実行してすべてのテストが通ることを確認してください。

コミットする際は、明確で説明的なコミットメッセージを使用してください。コミットメッセージは日本語で記述します。

### コミット前のチェックリスト

1. `/test`でテストがすべて通ることを確認
2. `npm run typecheck`で型エラーがないことを確認
3. `npm run build`でビルドエラーがないことを確認（必要に応じて）

## プルリクエストの作成

プルリクエストを作成する際は、`/pr`コマンドを使用してください。このコマンドは、変更内容を確認し、適切なコミットメッセージを作成し、プルリクエストを自動的に作成します。
