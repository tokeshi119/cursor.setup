# React開発ワークフロー

## 目的
Reactアプリの品質と一貫性を保つための最小指針を提供する。

## 必須ルール
- 関数コンポーネント + Hooks を標準とする
- レンダリング中の副作用は禁止（副作用はEffect/イベントで実行）
- Propsは不変、導出可能な状態は保持しない
- 単一責任で分割（UI / state / orchestration）

## Hooks運用
- `useEffect` の依存配列は正しく管理する
- 非同期Effectはキャンセル/ガードを入れる
- カスタムHooksで状態ロジックを再利用する
- `useMemo`/`useCallback` は計測して必要な場合のみ使う

## エラーハンドリング
- 画面単位でError Boundaryを配置する
- 失敗時のUIと再試行導線を用意する

## TypeScript
- Props/Stateは明示する
- `any` は原則禁止（必要なら範囲を限定する）

## アクセシビリティ（最低限）
- クリック可能要素はボタン/リンクを使う
- `aria-*` は必要最小限で正しく付与する

## テスト
- 重要なユーザー操作に対してUIテストを優先する
- 実装詳細より挙動を検証する

## コードスタイル
- 1ファイル1コンポーネント（小さなpure componentは例外可）
- 命名は用途が分かる動詞/名詞にする

## チェックリスト
- 状態の持ち方は最小か
- 副作用はEffect/イベントに閉じているか
- 依存配列は正しいか
- 主要操作のテストがあるか
