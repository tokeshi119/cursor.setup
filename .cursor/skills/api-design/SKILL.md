# API設計スキル

## 目的
REST API設計を一貫した基準で整理し、レビュー可能な形にする。

## 使うとき
- 新規API設計
- 既存APIの設計レビュー

## 入力
- ユースケース/画面/業務フロー
- 主要リソース（名詞）と関係
- 認証・認可の前提
- 想定クライアント（Web/モバイル/外部連携）
- 非機能要件（レート/キャッシュ/監査 など）

## 進め方
1) 目的と利用者を確認する
2) リソース/エンドポイント案を作成する
3) メソッド/命名/バージョン方針を決める
4) リクエスト/レスポンス/エラー/ページングを定義する
5) 認証・運用面（レート等）を確認する
6) チェックリストで漏れを確認する

## 出力（最小）
- 目的/対象範囲
- 主要リソース
- エンドポイント一覧
- リクエスト/レスポンス例
- エラー設計
- ページング方針
- 認証・認可方針
- 既知の懸念点
