# RulesとSkillsファイル作成プラン

## 前提確認

記事の内容を基に、以下の前提でRulesとSkillsファイルを作成します：

### Rules.mdの前提

1. **配置場所**: `.cursor/rules/` ディレクトリに配置
2. **性質**: 静的コンテキスト（すべての会話で常に参照される）
3. **構造**: WHY/WHAT/HOWの3要素を含む

- **WHAT**: 技術スタック、プロジェクト構造、コードベースの全体像
- **WHY**: プロジェクトの目的、各ディレクトリやコンポーネントの役割
- **HOW**: テスト、型チェック、ビルドなど、変更の是非を判断する手段

4. **長さ**: 300行未満（理想的には60行未満）
5. **内容**: 普遍的で適用可能な情報のみ（タスク固有の情報は別ファイルに分離）
6. **書き方**: 指示ではなく事実を書く（「必ず〜してください」より「このプロジェクトでは〜を使っている」）
7. **含めないもの**: コードスタイルガイドライン（リンターに任せる）、自動生成された内容
8. **段階的開示**: 詳細は別ファイルに分離し、参照ポイントだけを置く

### Skills.mdの前提

1. **配置場所**: `.cursor/skills/` ディレクトリに `SKILL.md` として配置
2. **性質**: 動的な機能とワークフロー（エージェントが必要に応じて読み込む）
3. **内容**: 

- **Custom commands**: `/` でトリガーされる再利用可能なワークフロー
- **Hooks**: エージェントのアクションの前後に実行されるスクリプト
- **Domain knowledge**: エージェントがオンデマンドで取り込める特定タスク向けの手順

4. **目的**: エージェントが実行できることの幅を広げる

## 実装内容

### 1. ディレクトリ構造の作成

- `.cursor/rules/` ディレクトリを作成
- `.cursor/skills/` ディレクトリを作成

### 2. Rulesファイルの作成

プロジェクト情報に基づいて、以下の内容を含む `rules.md` を作成：

- **WHAT**: TypeScript/JavaScript/Reactを使用したWebアプリケーションの構造
- **WHY**: プロジェクトの目的と各ディレクトリの役割
- **HOW**: npmコマンド、Vitestテスト、型チェックなどの検証方法
- コマンド一覧（npm run build, npm run test, npm run typecheck など）
- コードパターンへの参照（既存のコンポーネント構造など）

### 3. Skillsファイルの作成

一般的なワークフローを含む `SKILL.md` を作成：

- Gitワークフロー用のコマンド（例: `/pr` でプルリクエスト作成）
- コードレビュー用のコマンド（例: `/review`）
- テスト実行と反復改善のワークフロー
- 必要に応じてHooksの設定例

## 注意事項

- Rulesは簡潔に保ち、詳細は別ファイルに分離する方針
- Skillsは実際に使用するワークフローに基づいて段階的に追加
- コードスタイルガイドラインは含めず、リンター（Biome等）に任せる
- 自動生成ツール（`/init`等）は使用しない
