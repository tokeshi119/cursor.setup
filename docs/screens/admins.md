# 画面仕様: 管理者一覧/新規/編集

Last updated: 2026-02-02

## 1. 基本情報
- 画面ID: admins
- 画面名: 管理者一覧/新規/編集
- 目的/ゴール: 管理者の追加・更新・無効化・削除を行う
- 対象ユーザー/権限: 管理者（一般管理者はシステム管理者を閲覧不可）
- 添付スクショ/モック: 共有済み画像（2026-02-02, 一覧/編集）

## 2. 画面タイプ
- 一覧: yes
- 詳細: no
- 作成: yes
- 編集: yes
- 削除: yes（物理削除, 要確認）

## 3. 必要データ（最小）
- 参照テーブル: admins
- 主キー: admins.id
- 表示項目:
  - 管理ID（admins.id）
  - 状態（admins.is_active）
  - 管理者名（admins.name）
  - ログインID（admins.login_id）
  - 権限（admins.role）
  - メモ（admins.memo）
  - 操作（編集/削除）
- 編集項目:
  - 状態（有効/無効）
  - 管理者名
  - ログインID
  - 権限（システム管理者/一般管理者）
  - メモ
- 既定値/初期値:
  - 1ページ100件
  - 並び順: 管理ID昇順

## 4. 検索・フィルタ・ソート・ページング
- 検索条件: なし
- ソート条件: 管理IDの昇順固定
- ページング: page/limit（default=100, max=100）

## 5. 主要操作（ボタン/アクション）
- 新規登録
- 編集
- 削除（物理削除）

## 6. バリデーション/制約
- ログインID: 一意（admins.login_id）
- 権限:
  - admin = システム管理者
  - normal = 一般管理者
- 一般管理者の制限:
  - システム管理者は一覧に表示しない
  - システム設定は非表示
  - システムログは両者とも非表示

## 7. APIメモ（暫定）
- エンドポイント候補: GET/POST/PUT/DELETE /admins
- cognito_sub はサーバー側で設定（フロント入力なし）

## 8. 備考
- 削除は物理削除だが要確認
